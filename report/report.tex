\begin{abstract}
This report presents the final project for MAEG3080: training an image classifier from scratch on CIFAR-100 using PyTorch, without pretrained weights. The project uses a compact SE-ResNet-style architecture with squeeze-and-excitation blocks. We first train a short baseline with basic augmentations, then finetune (Stage~B) with RandAugment and a held-out validation split, obtaining \textbf{65.17\%} official test accuracy. We then apply a third stage (Stage~C): additional finetuning from the Stage~B checkpoint with RandAugment, Mixup/CutMix, and EMA at a lower learning rate. The \textbf{best} model achieves \textbf{68.14\%} on the official test set, evaluated with an independent \texttt{results.py} script as required by the project.
\end{abstract}

\section{Introduction}
The objective is to train a high-performing classifier on CIFAR-100 ($100$ classes, $32 \times 32$ RGB images). With no pretrained weights, performance depends on architecture choice, augmentation, and optimization. We prioritize a \emph{reproducible, efficient} pipeline (Google Colab + Google Drive checkpoints) and report official test accuracy from a dedicated evaluation script.

\section{Dataset}
CIFAR-100 contains $60{,}000$ images: $50{,}000$ for training and $10{,}000$ for testing. We normalize inputs with the standard channel statistics:
\[
\mu = (0.5071, 0.4867, 0.4408), \quad
\sigma = (0.2675, 0.2565, 0.2761).
\]
For finetuning, we hold out $10\%$ of the training labels as a stratified validation set (seed $42$); the baseline stage uses the full $50{,}000$ training images and evaluates on the test set each epoch for comparability with a historic short baseline.

\section{Methodology}
\subsection{Model Architecture}
We use a \textbf{small SE-ResNet} (\texttt{se\_resnet} in our codebase): residual stages with \textbf{squeeze-and-excitation} channel attention, \texttt{base\_width}${}=32$, \textbf{two} residual blocks per stage (\texttt{2$\times$2$\times$2$\times$2}), and global average pooling plus a linear classifier ($100$ classes). This design balances capacity and training time on commodity GPUs.

\subsection{Training Strategy}
We use a \textbf{three-stage} procedure:
\begin{itemize}[leftmargin=1.2em]
    \item \textbf{Stage A (baseline):} $4$ epochs on full training data, \textbf{basic} augmentation (crop, flip), \textbf{no} Mixup/CutMix, SGD (lr $0.1$), warmup $1$ epoch, cosine decay, test-set evaluation each epoch. Produces a $\sim$33\% test checkpoint used as initialization.
    \item \textbf{Stage B (finetune):} Load weights only (\texttt{--init-from}), \textbf{new} optimizer and cosine schedule. \textbf{RandAugment} ($N{=}2$, $M{=}9$), random erasing ($p{=}0.05$), label smoothing $0.05$, SGD with initial lr $0.05$, warmup $5$ epochs, $30$ epochs total. \textbf{EMA disabled.} Evaluation during finetuning uses the \textbf{validation} split; the \textbf{official test} is reserved for \texttt{results.py}.
    \item \textbf{Stage C (improvement):} Starting from the Stage~B \texttt{best.pt}, we run an additional $30$ epochs via \texttt{scripts/iterate\_from\_stageb\_stage\_c\_mixema.sh}: same SE-ResNet backbone, RandAugment ($N{=}2$, $M{=}9$), \textbf{Mixup} ($\alpha{=}0.2$) and \textbf{CutMix} ($\alpha{=}1.0$, mix probability $0.5$), label smoothing $0.05$, SGD with initial lr \textbf{$0.015$}, warmup \textbf{$3$} epochs, cosine decay, batch size $128$, gradient clipping $1.0$, and \textbf{EMA} (decay $0.999$). Train accuracy under mixup is not indicative of clean accuracy; we select by validation accuracy. Run name on Colab: \texttt{sota\_se\_resnet\_exp1}.
\end{itemize}
Training uses automatic mixed precision on CUDA where applicable (\texttt{torch.autocast}); checkpoints are written to disk under a configurable save directory.

\subsection{Implementation Details}
Implementation is in PyTorch (\texttt{train.py}, \texttt{model.py}, \texttt{utils.py}). Stages~A--B are run with \texttt{scripts/quick\_se\_resnet\_results.sh} (baseline if needed, finetune, then \texttt{results.py}). Stage~C is run separately (e.g.\ Colab notebook \texttt{CIFAR\_SOTA.ipynb}) by invoking \texttt{iterate\_from\_stageb\_stage\_c\_mixema.sh} with environment variables for paths, \texttt{RUN\_NAME}, \texttt{EPOCHS}, \texttt{LR}, and \texttt{WARMUP\_EPOCHS}. The \textbf{reported best test number} comes from \texttt{results.py} on the Stage~C \texttt{best.pt}.

\section{Discussion}
\begin{itemize}[leftmargin=1.2em]
    \item \textbf{Initialization matters:} Starting from the Stage~A weights gave a large jump over training the finetune recipe from scratch in the same wall-clock budget.
    \item \textbf{RandAugment + longer schedule:} Stronger augmentation and more epochs on top of a sane baseline dominated our earlier attempts with a much larger WRN trained from random initialization in limited time.
    \item \textbf{Stage~C on top of Stage~B:} Treating the $\sim$65\% model as a \emph{base} checkpoint and finetuning with Mixup, CutMix, and EMA at a lower learning rate yielded $\sim$3 percentage points on official test ($65.17\% \rightarrow 68.14\%$), with best validation $68.48\%$.
    \item \textbf{Trade-offs:} The SE-ResNet path is faster to iterate than a full Wide-ResNet Hydra ladder; the cost is a lower ceiling than very large models trained for hundreds of epochs on CIFAR-100.
\end{itemize}

\section{Conclusion}
We trained a compact SE-ResNet from scratch on CIFAR-100 using a baseline-plus-finetune workflow (Stages~A--B), then improved the model with a Stage~C finetune (Mixup, CutMix, EMA) from the Stage~B checkpoint. Our \textbf{best} model achieves \textbf{68.14\%} official test accuracy via \texttt{results.py}, with Stage~B alone at \textbf{65.17\%}. Future work could explore longer schedules or wider architectures if compute allows.
